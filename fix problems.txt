I am building a real-time location tracking web application using Node.js, Express, MySQL (Sequelize), and Socket.IO.

Current Situation:

* I have implemented a tracking system where a user generates a tracking link with a unique token.
* When the link is opened, the frontend requests location permission and sends GPS coordinates to the backend.
* The backend stores the location and emits real-time updates via Socket.IO.
* This works correctly when testing locally or on the same network.

Problem:

* When I open the tracking link on a mobile device, it shows a "Network Error".
* When I send the tracking link to friends or family members, it does not work properly, especially when they are on different networks (mobile data or different Wi-Fi).
* The location tracking either fails to start or does not send updates to the server.

Expected Outcome:

* The tracking link should work globally across different networks (not limited to localhost or same Wi-Fi).
* Any user who opens the tracking link and grants location permission should be able to share their live location successfully.
* The backend should receive location updates and broadcast them in real-time via Socket.IO.
* The system should behave similarly to location sharing in apps like WhatsApp or Google Maps.

Requirements:

   * Frontend request URLs

8. Ensure the solution is secure, scalable, and production-ready.

* Backend uses Express + Sequelize (MySQL)
